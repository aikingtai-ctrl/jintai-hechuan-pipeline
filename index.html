01 / 体系架构

三维管理体系

金泰负责基础设施与合规底座（专线接入远程算力、API 工程化、内容合规），河传负责创作与制片全流程。三个维度交叉构成矩阵式管理，任何制作节点都能定位到「哪一层、哪个部门、用什么接入」。

L1决策层
出品人 · 总制片人 · 项目总监
立项决策、预算终批、里程碑验收、成片终审与对外发布授权。持有 审核门①/⑥ 的最终签字权。
L2统筹层
制片主任 · 导演 · 技术总监（金泰）
排产调度、跨部门协调、质量把关、算力与档期分配。将决策拆解为部门任务，主持 审核门②–⑤ 评审。
L3部门层
各部门负责人 / 组长
部门内任务派发、交付物一审、日报与资产入库、对上汇报进度与风险。「责任到部门」的落点。
L4执行层
编剧 · 设计师 · AI 操作员 · 剪辑师 · 制片
具体制作：写圣经、出设计卡、跑 Seedance/Kling、远程达芬奇精剪、填表归档。「责任到人」的落点。

▤ 专线接入 · 分层级算力调度

河传本地工位经金泰专线直连远程算力集群，按项目/部门分配算力配额与优先级队列，制片日报回传占用与成本。

  • 接入方式专线（SD-WAN / 专用光纤）→ 金泰算力网关 → 分部门虚拟工位
  • 分层调度L2 技术总监按档期分配 GPU 池，L3 部门领用，L4 提交任务
  • 集群资源8×H100 + 4×RTX5090 渲染/生成节点 · 11 卡并行出镜
  • 编排层n8n 工作流编排：剧本→分镜→生成→合成自动流转
  • 成本回传每镜算力 / 时长 / 成本进入财务预算对照表

05 / 使用权管理 · 访问控制

核心生产资料的严格授权

ClaudeCode 与 Codex 是「影视圣经」的核心生产资料，不对学校学生（AI 操作员、剪辑师等 L4 执行层）开放。剧本工程与分镜/镜头设计等核心创作环节，由核心员工（剧本部/策划部/导演部/美术设计部）在受控工位使用 CC + Codex；学生只在沙箱内领用冻结后的圣经/分镜/素材，或经 API 网关代理提交生成任务——始终接触不到底层工具与密钥。采用「分级授权 · 最小权限 · 密钥零下发 · 网关单点管控 · 全量审计」的 RBAC 访问控制。

01

最小权限

按角色只授予完成本职所需的最小工具与数据范围，默认拒绝。

02

密钥零下发

ANTHROPIC_API_KEY / Codex 凭证只存金泰网关主机，绝不落地到学生/操作员机器。

03

网关单点管控

所有生成请求经 claude-bridge 网关 + BRIDGE_TOKEN 鉴权代理，本地拿不到底层工具。

04

全量审计

逐次调用留痕 who / when / what / cost，违规与超额可追溯、可回溯。
