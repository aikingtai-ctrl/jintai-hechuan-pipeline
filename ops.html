ACT I · 双线分流

两条通道，各司其职

交互式流量走专线，异步任务走 SD-WAN —— 人群划分是流量性质的自然结果。

点到点专线THE MAIN LINE · 交互通道 · 4–6ms
教师 + 博士/研究生 · 首批白名单 ≤30 人
  • 云桌面画面流：编码 Agent 环境（Codex 等）、云剪辑、调色、审片
  • 工具与密钥全部驻留金泰北京机房，人在河传只看画面、发键鼠
  • 每次会话实名绑定 + 全程审计
  • 低时延保障拖时间线、敲代码的真实手感
SD-WAN 互联网组网THE SECOND UNIT · 异步通道
全体学生 · 可大规模开放
  • 平台网页提交任务：国产大模型 API（GLM / DeepSeek / Qwen）
  • SEEDANCE 视频生成：提示词进 → 成片出，排队交付
  • 异步出片，公网抖动无影响
  • 全国产栈，干净、可写进合同、可放量
RED LINE · 合规红线

本片含一条不可逾越的红线

海外编码工具绝不以"发 API key"的方式提供给校方人员——那等同向第三方转售海外模型通道。 唯一合规形态是云桌面：工具装在金泰机房、海外调用从金泰发起、密钥不出机房、专线上只跑画面流。 对外合同与宣传写"智能编码 Agent 开发环境"，可承诺名单 = Codex（香港实体企业渠道）+ 国产编码模型。

ACT II · 一对一对接

三层治理，学院对位

固化成制度，而不是"谁嗓门大谁用得多"。需求走平台工单，不走口头。

金泰负责人 + 学校分管校领导：定方向、定资源、定结算。制度化立项，对冲人事变动。

每周 · WEEKLY

每个学院一组：金泰对接人（制片/客户成功）⇌ 学院教学秘书/专业负责人。需求单制，周例会。

故障、账号、机房环境的日常通道。专线与平台的健康状态双向可见。

ACT III · 数据不落地
